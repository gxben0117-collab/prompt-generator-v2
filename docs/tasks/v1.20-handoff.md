## 後續要做

1. 角色卡資料拆檔

目前 `src/data.js` 已很大。後續建議把角色卡資料拆成：

```text
src/data/roles/historicalNovel.js
src/data/roles/dynasty.js
src/data/roles/wuxia.js
src/data/roles/xianxia.js
src/data/roles/travel.js
src/data/roles/modern.js
src/data/roles/darkRoyal.js
src/data/roles/fairyGarden.js
```

再由 `src/data.js` 統一匯入，降低維護難度。

2. 角色卡管理 UI

後續可以在 UI 加：

- 角色卡數量統計。
- 依分類顯示卡片數。
- 搜尋關鍵字高亮。
- 最近使用角色卡。
- 收藏角色卡。

3. 動作語言品質抽驗

下一輪建議抽驗：

- 坐姿是否保持身體比例。
- 撫琴、泡茶、持相機是否會遮臉。
- 持刀、持劍、持弓是否過度戰鬥姿勢。
- 現代街拍是否仍保留電影主視覺，而不是普通生活照。

4. 仙俠 / 古裝陸劇補強

可後續補：

- 更多 `長相思` 類型卡。
- 更多大唐飛天變體。
- 更多神話女性角色。
- 古裝陸劇常見場景，如宮宴、花朝、上元燈會、雨夜長街、山谷醫館。

5. Build chunk 優化

目前 Vite build 有 chunk 超過 500KB 的警告，原因是資料量增加。不是錯誤，但後續拆資料檔或 lazy loading 後可改善。

## 注意事項

- 不要直接手改 `index.html`，它是 build 成品。
- 主要修改入口仍是 `src/`、`doc/`、`tests/`。
- 固定母板不要亂改；這次 v1.20 沒有改母板，只新增角色卡資料與驗證。
- 角色卡新增時優先使用 `createCuratedRoleProfile(...)`，不要手寫散亂 profile。
