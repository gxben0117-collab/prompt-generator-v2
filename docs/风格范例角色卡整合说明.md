## 技术实现

### 文件结构
```
src/
├── profiles/
│   ├── styleAnalysisProfiles.js  # 新创建的角色卡定义文件
│   └── index.js                   # 更新：导出新的角色卡
└── data.js                        # 更新：整合到系统中
```

### 数据格式
每个角色卡包含以下完整字段：
- `id`: 唯一标识符
- `title`: 显示标题
- `parentCategory`: 父分类（匹配系统分类）
- `series`: 系列名称
- `themeHint`: 主题提示
- `aliases`: 别名数组
- `identity`: 身份描述
- `palette`: 配色方案
- `costumeCore`: 核心服装描述
- `jewelry`: 首饰配饰
- `prop`: 道具
- `place`: 场景位置
- `foreground`: 前景元素
- `midground`: 中景人物
- `background`: 背景元素
- `action`: 动作描述
- `sceneAction`: 场景动作
- `sceneLighting`: 场景光线
- `lighting`: 灯光设置
- `atmosphere`: 氛围描述
