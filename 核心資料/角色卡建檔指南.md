## 角色卡資料結構

```json
{
  "id": "唯一識別碼",
  "name": "角色卡名稱",
  "sub": "子分類名稱",
  "icon": "🎭",
  "scene": "場景描述（英文）",
  "outfit": "服裝描述（英文）",
  "makeup": "妝容描述（英文或中文）",
  "prop": "動作與道具描述（英文）",
  "comp": "構圖描述（英文）",
  "mk": "妝容代碼（可選）",
  "tpl": "模板類型",
  "ang": "鏡頭角度代碼",
  "ratio": "圖片比例代碼",
  "lens": "鏡頭焦段代碼",
  "atm": "氛圍代碼",
  "camLang": "鏡頭語言代碼",
  "light": "燈光風格代碼"
}
```

## 欄位說明與規則

### 1. **id**（必填）
- **用途**：唯一識別碼
- **格式**：`前綴_數字`（例如：`hr_001`, `dq_001`）
- **規則**：
  - 同一系列使用相同前綴
  - 數字從 001 開始遞增
  - 不可重複

### 2. **name**（必填）
- **用途**：角色卡顯示名稱，也是預設主題
- **格式**：中文或英文
- **規則**：
  - ✅ **可以**：具體的服裝、顏色、場景、劇情、動作、職業、人物名字
  - ❌ **禁止**：氣質描述、長相描述、性格描述、外表描述
  
**禁止詞彙**：
- 氣質：機靈俏皮、英氣颯爽、溫婉知性、華貴典雅、才女風範、優雅、高貴、嫵媚
- 長相：美貌、絕色、美麗、傾城、沉魚落雁、閉月羞花
- 性格：溫柔、殘酷、冷豔、熱情、活潑

**正確範例**：
- ✅ 黃蓉 · 桃花島初遇郭靖
- ✅ 魅魔 · 華麗轉身
- ✅ 暗夜女王 · 血月城堡
- ✅ 三國江南小喬

**錯誤範例**：
- ❌ 黃蓉 · 少女時期，機靈俏皮
- ❌ 江東絕色 · 小喬
- ❌ 溫柔救贖 · 難哄

### 3. **sub**（必填）
- **用途**：子分類名稱，顯示在角色卡下方
- **格式**：中文
- **範例**：金庸武俠、暗黑哥德、仙俠修真

### 4. **icon**（必填）
- **用途**：角色卡圖示
- **格式**：單一 emoji
- **範例**：🎭, 👑, ⚔️, 🌸, 🔥

### 5. **scene**（必填）
- **用途**：場景描述，包含動作、地點、環境、天氣
- **語言**：英文
- **規則**：
  - ✅ **可以**：動作、地點、環境背景、特效元素、天氣、光線
  - ❌ **禁止**：角色外觀、服裝、妝容、氣質描述
  
**禁止詞彙**：
- 動作氣質：elegantly（優雅地）, gracefully（優美地）, gently（溫柔地）
- 外觀描述：beautiful face, gorgeous appearance, stunning look

**正確範例**：
```
Moonlit courtyard, guqin resting on lap, plum blossom tree shadows, 
soft moonlight filtering through branches, peaceful night atmosphere
```

**錯誤範例**：
```
❌ sitting elegantly with guqin (包含氣質描述)
❌ beautiful woman in moonlit courtyard (包含外觀描述)
```

### 6. **outfit**（必填）
- **用途**：服裝描述
- **語言**：英文
- **規則**：
  - ✅ **可以**：衣服款式、材質、風格、顏色、吊飾、首飾、頭飾
  - ❌ **禁止**：氣質描述、穿著方式的氣質形容
  
**正確範例**：
```
white ruqun dress, light cyan outer robe, simple hair ornaments, 
silver hairpin with jade pendant
```

**錯誤範例**：
```
❌ elegant gown (elegant 用於服裝本身可以，但要避免 "wearing elegantly")
```

### 7. **makeup**（選填）
- **用途**：妝容描述
- **語言**：英文或中文皆可
- **規則**：
  - 僅描述表面妝容效果（眼影、眼線、腮紅、唇彩、髮型）
  - 不可描述改變骨相的內容
  
**正確範例**：
```
dramatic red lips, long black hair with natural waves, defined eye makeup
```
或
```
魅魔妝容：深紫色煙燻眼妝，紅唇，黑色長髮
```

### 8. **prop**（必填）
- **用途**：動作與道具描述
- **語言**：英文
- **規則**：
  - 描述人物動作、姿勢、持有道具
  - ❌ **禁止**：氣質動作描述（sitting elegantly, standing gracefully）
  
**正確範例**：
```
a woman sitting on throne holding ornate skull staff, 
hand resting near face, direct eye contact with camera
```

**錯誤範例**：
```
❌ sitting elegantly on throne
❌ standing gracefully with staff
❌ walking gracefully through garden
```

### 9. **comp**（必填）
- **用途**：構圖描述
- **語言**：英文
- **規則**：描述畫面構圖、人物位置、視角
  
**範例**：
```
vertical medium shot composition waist up, realistic head-to-body proportion, 
natural face-neck-shoulder alignment, complete costume readability
```

### 10. **mk**（選填）
- **用途**：妝容代碼，對應 `妝容模組.json` 中的定義
- **格式**：字串代碼
- **範例**：`succubus_alluring`, `xianxia`, `gothic`
- **注意**：如果填寫了 `makeup` 欄位，則優先使用 `makeup`；如果 `makeup` 為空，則使用 `mk` 代碼查找預設妝容

### 11. **tpl**（必填）
- **用途**：模板類型
- **固定值**：`default`

### 12. **ang**（必填）
- **用途**：鏡頭角度代碼
- **可選值**：
  - `quan`：全身鏡頭
  - `banshen`：半身鏡頭
  - `huanjing`：環境人像

### 13. **ratio**（必填）
- **用途**：圖片比例代碼
- **可選值**：
  - `r_11`：1:1 正方形
  - `r_23`：2:3 直向
  - `r_34`：3:4 直向人像
  - `r_43`：4:3 橫向
  - `r_916`：9:16 手機螢幕
  - `r_cinema`：2.39:1 電影橫幅

### 14. **lens**（必填）
- **用途**：鏡頭焦段代碼
- **固定值**：`l_50`（50mm 鏡頭）

### 15. **atm**（必填）
- **用途**：氛圍代碼
- **可選值**：
  - `at_clear`：清晰明亮
  - `at_moody`：黑暗情緒
  - `at_misty`：柔和霧氣

### 16. **camLang**（必填）
- **用途**：鏡頭語言代碼
- **可選值**：
  - `cl_magazine`：時尚雜誌風格
  - `cl_fashion`：編輯攝影風格
  - `cl_documentary`：紀錄片風格

### 17. **light**（選填）
- **用途**：燈光風格代碼
- **可選值**：
  - `ls_natural`：自然日光
  - `ls_cinematic`：電影級照明
  - `ls_documentary`：紀錄片照明

## 禁止詞彙清單

### 中文禁止詞彙
**氣質描述**：
- 機靈俏皮、英氣颯爽、溫婉知性、華貴典雅、才女風範
- 優雅、高貴、嫵媚、端莊、嬌媚、冷豔
- 溫柔、殘酷、熱情、活潑、文靜

**長相描述**：
- 美貌、絕色、美麗、傾城、沉魚落雁、閉月羞花
- 俊美、清秀、嬌美、豔麗

**性格描述**：
- 溫柔、冷酷、熱情、活潑、內向、外向

### 英文禁止詞彙
**氣質動作描述**：
- sitting elegantly（優雅地坐著）
- standing gracefully（優美地站立）
- walking gracefully（優美地行走）
- moving gently（溫柔地移動）
- posing elegantly（優雅地擺姿勢）

**外觀描述**：
- beautiful face（美麗的臉）
- gorgeous appearance（華麗的外表）
- stunning look（驚艷的外貌）
- attractive features（迷人的特徵）

**性格描述**：
- gentle personality（溫柔的性格）
- elegant demeanor（優雅的舉止）
- graceful manner（優美的態度）

## 建檔流程

### 步驟 1：確定角色主題
1. 確定角色所屬的父分類（例如：金庸武俠、暗黑哥德）
2. 確定角色名稱（避免使用禁止詞彙）
3. 設計 3-6 個不同場景的角色卡

### 步驟 2：生成角色卡 ID
- 格式：`前綴_數字`
- 同一系列使用相同前綴
- 範例：`hr_001`, `hr_002`, `hr_003`（黃蓉系列）

### 步驟 3：撰寫場景描述
1. 描述地點、環境、天氣、光線
2. 描述動作（避免氣質形容詞）
3. 使用英文撰寫
4. 檢查是否包含禁止詞彙

### 步驟 4：撰寫服裝描述
1. 描述衣服款式、材質、顏色
2. 描述配飾、首飾、頭飾
3. 使用英文撰寫
4. 避免氣質形容詞

### 步驟 5：撰寫妝容描述
1. 描述妝容風格（眼妝、唇妝、髮型）
2. 可使用英文或中文
3. 僅限表面妝容效果

### 步驟 6：撰寫動作與道具
1. 描述人物姿勢、動作
2. 描述持有的道具
3. 使用英文撰寫
4. **重點檢查**：避免 elegantly, gracefully, gently 等氣質副詞

### 步驟 7：選擇技術參數
- 鏡頭角度：quan（全身）/ banshen（半身）/ huanjing（環境）
- 圖片比例：r_916（手機）/ r_34（人像）/ r_23（直向）等
- 氛圍：at_clear（明亮）/ at_moody（黑暗）/ at_misty（霧氣）
- 鏡頭語言：cl_documentary（紀錄片）/ cl_fashion（時尚）/ cl_magazine（雜誌）

### 步驟 8：驗證與檢查
使用檢查腳本驗證：
```bash
node check_forbidden_words.mjs
```

### 步驟 9：加入 index.html
1. 找到對應的分類（CATS 陣列）
2. 在 `entries` 陣列中新增角色卡
3. 執行建置：`npm run build`

## 常見問題 FAQ

### Q1：什麼時候可以使用 "elegant"？
**A**：
- ✅ 可以：描述服裝本身（elegant gown = 優雅的禮服）
- ❌ 禁止：描述動作方式（sitting elegantly = 優雅地坐著）

### Q2：makeup 欄位可以留空嗎？
**A**：可以。如果留空，系統會：
1. 優先使用 `mk` 代碼查找預設妝容
2. 如果 `mk` 也為空，則使用「依主題讓AI設計」

### Q3：如何選擇鏡頭角度？
**A**：
- `quan`（全身）：需要展示完整服裝，從頭到腳
- `banshen`（半身）：腰部以上，適合展示上半身細節
- `huanjing`（環境）：人物在更大的場景中，強調環境氛圍

### Q4：如何選擇圖片比例？
**A**：
- `r_916`：手機螢幕，適合社群媒體分享
- `r_34`：直向人像，適合半身照
- `r_23`：高直向，適合全身照
- `r_11`：正方形，適合 Instagram
- `r_43`：橫向，適合風景式構圖
- `r_cinema`：電影橫幅，適合寬螢幕效果

### Q5：如何批量檢查角色卡是否違規？
**A**：執行檢查腳本：
```bash
node check_forbidden_words.mjs
```

### Q6：可以使用中文描述嗎？
**A**：
- `name`：可以使用中文
- `sub`：必須使用中文
- `scene`, `outfit`, `prop`, `comp`：建議使用英文
- `makeup`：中文或英文皆可
