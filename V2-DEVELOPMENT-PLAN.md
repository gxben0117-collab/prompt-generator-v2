## 🎯 v2.0 目標與限制

### 核心目標
✅ **模組化分類管理** - 每個分類獨立定義，易於維護  
✅ **精簡到 50 個核心分類** - 從 115 個可用分類中精選不相似的 50 個  
✅ **角色卡外部化** - JSON 格式儲存，批次更新更容易  
✅ **保持 100% 向下相容** - 所有功能完整保留  

### 明確限制
❌ **不包含已刪除的 6 個分類**  
❌ **不超過 50 個分類** - 用戶明確要求上限  
❌ **排除相似分類** - 避免重複概念（如多個「江南水鄉」變體）

### 2.2 分類模組標準格式

每個分類檔案包含：

```javascript
// src/categories/01-eastern-costume/hongloumeng.js
export default {
  id: 'hongloumeng',
  name: '紅樓夢／金陵十二釵',
  parentCategory: '歷史小說名著人物',
  
  // 分類特色
  visualStyle: {
    palette: '玫紅、湖綠、象牙白與金色',
    costumePreference: '明清仕女服飾、刺繡雲肩、玉飾',
    actionPattern: '庭院倚欄、撫琴讀書、賞花對月',
    propRules: '團扇、詩卷、花枝、玉佩',
  },
  
  // 推斷規則
  classificationRules: {
    keywords: ['紅樓夢', '金陵', '林黛玉', '薛寶釵', '大觀園'],
    priority: 10,
    exclusions: ['武俠', '仙俠', '暗黑'],

  // 預設場景
  defaultScenes: {
    lighting: 'golden hour soft light, silk fabric highlights, garden reflection',

  },
  
  // 參考角色卡（指向 JSON 檔案）
  profilesFile: 'hongloumeng-profiles.json',
};
```

## ⚠️ 風險評估

### 高風險項目
1. **角色卡遺失** - 未精選的 65 個分類中的 102 張角色卡需要處理
   - **緩解**: 先備份，由用戶決定保留/刪除/合併
   
2. **分類推斷失效** - 新的 50 分類系統可能影響自動推斷
   - **緩解**: 保留舊系統作為 fallback

### 中風險項目
3. **10 層服裝邏輯提取錯誤** - 可能遺漏細節
   - **緩解**: 逐層驗證，對比原始輸出
